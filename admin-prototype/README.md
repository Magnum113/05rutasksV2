# Admin Prototype: Tasks Marketplace

Интерактивный HTML-прототип админки фичи "Задания" по PRD.

## Что внутри

- `index.html` — оболочка и экраны админки.
- `styles.css` — визуальная система и адаптив.
- `mock-data.js` — демо-данные задач, статусов пользователей и очков.
- `app.js` — логика интерфейса, фильтров, форм, валидаций, модалок и CSV-экспорта.

## Как открыть

1. Откройте файл `/Users/kadimagomedov/Documents/05ruTasksV2/admin-prototype/index.html` в браузере.
2. Локальный сервер не обязателен, но можно использовать любой static server для удобства.

## Покрытие требований PRD

- Полный flow роли маркетолога:
  - список заданий;
  - создание/редактирование задания;
  - экран выполнения заданий пользователями;
  - экран пользователей и очков активности.
- Поддержаны все `task_type` 1-8.
- Поддержаны награды: `bonus`, `promocode`, `activity_points`, а также комбинации.
- Реализованы валидации:
  - `active_from < active_to`;
  - обязательный purchase-период для purchase-заданий;
  - уникальность `task_code`;
  - поля промокода и очков активности;
  - валидация URL для visit-задач и внешних target-ссылок;
  - предупреждение для `status_mode=force`.
- Реализована симуляция статусов:
  - `upcoming`, `active`, `completed`, `reward_claimed`, `expired`;
  - 7-дневное окно claim после завершения задания.
- Сортировка задач: приоритет (desc), затем новизна (created_at desc).
- Экспорт CSV:
  - список задач;
  - выполнение пользователей по заданию.

## Демо-сценарии

В экране "Создать задание" доступны быстрые кнопки:

- `visit_url + promocode + activity_points`
- `purchase-задача + fixed purchase period`

Дополнительно можно вручную проверить:

- ошибку при пустом `task_code`;
- ошибку при `active_to <= active_from`;
- ошибку при незаполненных полях промокода;
- фильтр списка задач по наличию `activity_points`;
- пользователей в статусах `completed` в 7-дневном окне claim и `expired` после окна.

## Ограничения прототипа

- Нет backend-интеграции и реальных API.
- Нет интеграции с 1С.
- Нет production-авторизации.
- Статусы и данные в интерфейсе демонстрационные (на mock-данных).
